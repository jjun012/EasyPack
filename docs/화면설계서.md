# 화면 설계서

- 프로젝트명: EasyPack
- 플랫폼: React Native (Expo)
- 작성일: 2026

---

## 1. 화면 흐름도

```
앱 실행
    │
    ▼
[SplashScreen] ── 토큰 있음 ──► [HomeScreen]
    │
  토큰 없음
    │
    ▼
[LoginScreen] ◄─────────────────────────────┐
    │                                        │
  로그인 성공                              로그아웃
    │                                        │
    ▼                                        │
[HomeScreen] ────────────────────────────────┘
    │
    ├── 물품촬영 ──► [CameraScreen] ──► [DetectedInfoScreen]
    │
    ├── 커뮤니티 ──► [CommunityScreen]
    │                   └── [나라Board] ──► [ReviewScreen]
    │
    ├── 수하물정보 ──► [InfoScreen] ─► [InfoScreen2] ─► [InfoScreen3]
    │
    ├── 항공사정보 ──► [AirlineInfoScreen]
    │
    └── 설정 ──► [SettingsScreen]
```

## 2. 화면 상세 설계

---

### 2.1 SplashScreen

**목적**: 앱 시작 시 로그인 상태 확인

**화면 구성**
- EasyPack 로고 (중앙)
- 로딩 인디케이터 (하단)

**로직**
```
AsyncStorage에서 accessToken 확인
  ├── 토큰 있음 → HomeScreen으로 이동
  └── 토큰 없음 → LoginScreen으로 이동
```

**수정 사항**
- 기존 버그: `access_token` → `accessToken` 으로 키 통일

---

### 2.2 LoginScreen

**목적**: 사용자 로그인

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 로고 이미지 | 상단 중앙 |
| 아이디 입력 | TextInput |
| 비밀번호 입력 | TextInput (secureTextEntry) |
| 로그인 버튼 | 파란색 버튼 |
| 회원가입 링크 | 하단 텍스트 |
| 에러 메시지 | 로그인 실패 시 표시 |
| 로딩 모달 | 요청 중 표시 |

**API 연동**
- POST /api/auth/login
- 성공 시: token, user 정보 AsyncStorage 저장 후 HomeScreen 이동

---

### 2.3 RegisterScreen

**목적**: 3단계 회원가입

**Step 1 - 약관 동의**
| 항목 | 필수 여부 |
|------|----------|
| 이용 약관 동의 | 필수 |
| 개인정보 수집 및 이용 동의 | 필수 |
| 개인정보 이용 안내 | 선택 |
| 마케팅 수신 동의 | 선택 |
| 전체 동의 | - |

**Step 2 - 계정 정보**
| 항목 | 검증 |
|------|------|
| 아이디 | 필수 |
| 닉네임 | 필수 |
| 비밀번호 | 필수 |
| 비밀번호 확인 | 비밀번호 일치 확인 |

**Step 3 - 여행 정보**
| 항목 | 선택지 |
|------|--------|
| 여행지 | 베트남, 미국, 일본, 태국, 필리핀 |
| 항공사 | 대한항공, 아시아나항공, 제주항공, 티웨이항공, 진에어항공 |

**API 연동**
- POST /api/auth/register
- 성공 시: LoginScreen으로 이동

---

### 2.4 HomeScreen

**목적**: 메인 화면, 주요 기능 진입점

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 설정 버튼 | 우측 상단 유저 아이콘 |
| EasyPack 로고 | 상단 |
| 여행지-항공사 배지 | 예: JAPAN - 대한항공 ✈️ |
| 메뉴 3개 | 물품촬영, 커뮤니티, 수하물정보 |
| 항공사 수하물 배너 | 내 항공사 규정 확인 버튼 |
| 여행지 배경 이미지 | 여행지에 맞는 이미지 |
| 날씨 정보 | 여행지 현재 날씨 |

**API 연동**
- GET /api/auth/user/me (화면 진입 시)
- OpenWeatherMap API (날씨)

---

### 2.5 CameraScreen

**목적**: 물품 촬영 및 AI 분석

**화면 구성**
| 상태 | 화면 |
|------|------|
| 기본 | 카메라 뷰 + 촬영 버튼 |
| 분석 중 | "분석 중..." 텍스트 + 로딩 |
| 인식 실패 | 수동 입력 텍스트박스 + 제출 버튼 + 재촬영 버튼 |

**로직**
```
촬영 버튼 클릭
    │
    ▼
사진 촬영
    │
    ▼
FastAPI /predict 호출 (이미지 + country + airline)
    │
    ├── 인식 성공 ──► DetectedInfoScreen으로 이동 (label, description, imageUri)
    └── 인식 실패 ──► 수동 입력 UI 표시
                          │
                     텍스트 입력 후 제출
                          │
                     FastAPI /predict/text 호출
                          │
                     DetectedInfoScreen으로 이동
```

**수정 사항**
- 기존: AsyncStorage에서 travelDestination, airline 읽음 (저장 시점 문제)
- 개선: API로 최신 사용자 정보 조회 후 사용

---

### 2.6 DetectedInfoScreen

**목적**: AI 분석 결과 표시

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 물품명 | 인식된 물품 이름 (대문자) |
| 촬영 이미지 | 찍은 사진 (수동입력 시 없음) |
| 안내 사항 제목 | "안내 사항" |
| 규정 설명 | AI 분석 결과 텍스트 |
| 돌아가기 버튼 | CameraScreen으로 복귀 |

---

### 2.7 CommunityScreen

**목적**: 나라별 게시판 목록

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 제목 "커뮤니티" | 상단 |
| 여행기 섹션 | 5개 나라 게시판 리스트 |
| 각 게시판 항목 | 나라명 + 최신 게시글 미리보기 |
| 인기글 섹션 | 좋아요 수 기준 상위 5개 |

**API 연동**
- GET /api/community/posts/popular
- GET /api/community/posts/country/{country} (미리보기용 최신 1개)

**수정 사항**
- 기존: AsyncStorage에서 로컬 데이터 읽음
- 개선: 백엔드 API 연동

---

### 2.8 BoardScreen (공통 컴포넌트)

**목적**: 나라별 게시판 (5개 게시판을 1개 컴포넌트로 통합)

**Props**
```javascript
{
  country: '일본' | '미국' | '베트남' | '필리핀' | '태국',
  boardTitle: '일본 게시판'
}
```

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 헤더 | 뒤로가기 + 게시판 제목 + 글쓰기 버튼 |
| 게시글 목록 | FlatList |
| 게시글 아이템 | 프로필, 닉네임, 별점, 제목, 내용, 좋아요, 댓글 |

**수정 사항**
- 기존: JapanBoard, USABoard 등 5개 파일 (완전 중복)
- 개선: BoardScreen 1개 컴포넌트 + country props

---

### 2.9 ReviewScreen

**목적**: 게시글 작성

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 나라 선택 버튼 | 5개 나라 중 선택 |
| 별점 선택 | ★ 1~5 |
| 제목 입력 | TextInput |
| 내용 입력 | TextInput (multiline) |
| 제출 버튼 | |

**API 연동**
- POST /api/community/post

---

### 2.10 InfoScreen / InfoScreen2 / InfoScreen3

**목적**: 수하물 규정 안내 (3페이지)

| 화면 | 내용 |
|------|------|
| InfoScreen | 반입 금지 물품 |
| InfoScreen2 | 제한적 기내 반입 물품 |
| InfoScreen3 | 위탁 수하물 제한 물품 |

**공통 구성**
- 아코디언 방식으로 항목 펼치기/접기
- 이전/다음 페이지 이동 버튼

---

### 2.11 AirlineInfoScreen

**목적**: 내 항공사 수하물 규정 + 초과 요금 계산

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 항공사 로고 | |
| 항공사명 | |
| 수하물 규정 카드 | 위탁/기내 수하물 규정 |
| 무게 입력 | TextInput (숫자) |
| 계산 버튼 | |
| 결과 카드 | 예상 초과 요금 (애니메이션) |

**API 연동**
- GET /api/auth/user/me (항공사 정보)

---

### 2.12 SettingsScreen

**목적**: 회원정보 수정

**화면 구성**
| 컴포넌트 | 설명 |
|----------|------|
| 프로필 이미지 | 기본 아이콘 |
| 아이디 | 읽기 전용 |
| 비밀번호 | 변경 시만 입력 |
| 닉네임 | 수정 가능 |
| 여행 국가 | Picker |
| 항공사 | Picker |
| 수정 버튼 | |

**API 연동**
- GET /api/auth/user/me (화면 진입 시)
- PUT /api/auth/user/update (수정 버튼)

```javascript
│   └── RegisterScreen
└── DrawerNavigationRoutes (Drawer)
    ├── HomeScreen
    ├── CameraScreen (Stack)
    │   └── DetectedInfoScreen
    ├── CommunityScreen
    ├── InfoScreen
    ├── InfoScreen2
    ├── InfoScreen3
    ├── AirlineInfoScreen
    ├── SettingsScreen
    ├── ReviewScreen
    ├── JapanBoard     ┐

```

## 4. 토큰 관리 규칙

| 항목 | 키 이름 | 저장 시점 |
|------|---------|----------|
| JWT 토큰 | `accessToken` | 로그인 성공 시 |
| 유저 ID | `userId` | 로그인 성공 시 |

> 전체 코드에서 `accessToken` 으로 통일 (기존 `access_token` 버그 수정)
