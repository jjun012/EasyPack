## 3. 배포 구성

### 3.1 배포 환경
| 서비스 | 플랫폼 | 무료 한도 |
|--------|--------|----------|
| Spring Boot | Railway | 월 $5 크레딧 |
| FastAPI | Render | 무료 (슬립 있음) |
| PostgreSQL | Supabase | 500MB |
| 앱 빌드 | Expo EAS | 월 30빌드 |

### 3.2 환경변수 관리
**Spring Boot (application.yml)**
```yaml
spring:
  datasource:
    url: ${DB_URL}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
jwt:
  secret: ${JWT_SECRET}
```

**FastAPI (.env)**
```
GEMINI_API_KEY=
DB_HOST=
DB_USER=
DB_PASSWORD=
DB_NAME=
```

## 4. 보안 설계

| 항목 | 방식 |
|------|------|
| 비밀번호 | BCrypt 암호화 저장 |
| API 인증 | JWT Bearer Token |
| JWT 만료 | 1시간 |
| 민감정보 | 환경변수로 관리 (.env, Railway/Render 환경변수) |
| CORS | 허용 출처 명시적 설정 |

## 5. 기존 대비 개선 요약

| 항목 | 기존 | 개선 |
|------|------|------|
| DB | AWS RDS MySQL (유료) | Supabase PostgreSQL (무료) |
| AI | YOLO (낮은 정확도) | Gemini Vision (높은 정확도) |
| 보안 | DB 비밀번호 코드에 노출 | 환경변수로 분리 |
| 커뮤니티 | AsyncStorage (로컬) | 백엔드 API 연동 |
| 게시판 | 5개 중복 파일 | 컴포넌트 1개 통합 |
| 자동로그인 | 토큰 키 불일치 버그 | accessToken으로 통일 |
| 배포 | EC2 (유료) | Railway + Render (무료) |
